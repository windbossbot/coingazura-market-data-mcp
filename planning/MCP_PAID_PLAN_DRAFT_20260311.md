# [MCP] Paid Plan Draft

Updated: 2026-03-11

Status:

- planning only
- no pricing change should be made yet

## Free Tier Position

- local stdio install
- read-only tool set
- fixed-host live resource fetch
- evaluation and lightweight agent usage

## Paid Tier Candidate

Only introduce paid once these are real:

- hosted endpoint
- clearer freshness guarantees
- richer request budget
- higher-value premium summaries
- support boundary

## Possible Tier Shapes

### Free

- local install
- current 3 tools
- best-effort usage

### Pro

- hosted endpoint access
- higher request allowance
- stronger freshness and uptime wording
- premium stable/risk briefs

### Team

- hosted endpoint
- higher limits
- support response target
- team-facing deployment guidance

## Minimum Proof Before Pricing

- hosted architecture note exists
- security model for hosted mode exists
- support scope exists
- billing language does not outrun real capability

## Messaging Rule

- sell trust and reliability, not inflated feature count
- keep the free version useful
- make the paid version about delivery guarantees and support, not hype
