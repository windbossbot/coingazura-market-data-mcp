# [MCP] Paid Readiness Notes

Updated: 2026-03-11

Status:

- current pricing should remain `Free`
- paid conversion is preparation-only for now

## Goal

Define what should change before a paid tier is credible.

## Do Not Sell Yet

- the current server is local stdio first
- remote hosted endpoint is not provided yet
- no freshness SLA is promised yet
- support and response commitment are not defined yet

## Paid Tier Should Mean Something Concrete

Possible paid differentiators:

- hosted remote endpoint
- higher request limits
- better freshness guarantees
- richer history windows
- premium risk brief outputs
- priority support

## Minimum Requirements Before Paid Conversion

1. Hosted delivery path defined
2. Clear support boundary defined
3. Freshness and reliability wording defined
4. Marketplace copy split into free vs paid value
5. Security policy updated for hosted operation if remote serving is added

## Safe Free Positioning For Now

Recommended current stance:

- free read-only install
- security-first narrow scope
- good for evaluation, local use, and agent experimentation

## Risks If Paid Too Early

- users may expect uptime guarantees that do not exist
- users may expect broader coverage than current scope
- users may expect direct exchange or wallet actions
- score can fall if pricing outruns trust and clarity

## Next Paid-Prep Deliverables

- hosted roadmap note
- free vs paid comparison table
- support and SLA draft
- marketplace pricing copy draft
